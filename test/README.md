# 🧪 m.weibo.cn API 测试脚本

## 📋 项目说明

这是一个独立的测试脚本，用于测试微博移动端 API (`m.weibo.cn`) 的数据获取功能。它从主项目中抽离了核心逻辑，方便快速调试和验证 API 接口。

### ✨ 主要功能

- ✅ 获取指定 UID 用户的微博数据
- ✅ 解析微博文本、图片、视频、LivePhoto
- ✅ 显示用户基本信息（昵称、粉丝数等）
- ✅ 自动处理分页和 since_id
- ✅ 智能错误检测（验证码、频率限制等）
- ✅ 随机延时模拟人类操作，降低被封风险

## 🔑 如何获取 Cookie

### 方法一：浏览器开发者工具（推荐）

1. **打开浏览器** (Chrome/Edge)
2. **访问** https://m.weibo.cn/
3. **登录微博账号**
4. **按 F12** 打开开发者工具
5. **切换到 Network (网络) 标签**
6. **刷新页面** (F5)
7. **点击任意请求** (建议选择第一个请求)
8. **找到 Request Headers → Cookie**
9. **复制整个 Cookie 值**

### 方法二：JavaScript 快速获取

在浏览器控制台 (Console) 中执行：

```javascript
document.cookie
```

然后复制输出的内容。

## ❌ 常见问题

### 问题 1: API 返回 `ok=-100`

**原因**: 触发验证码验证或反爬机制

**解决方案**:
1. 等待 30-60 分钟后重试
2. 重新获取新的 Cookie
3. 更换网络环境（如使用手机热点）
4. 增加延时时间

### 问题 2: HTTP 403 Forbidden

**原因**: Cookie 完全失效或账号被限制

**解决方案**:
1. 清除浏览器缓存并重新登录
2. 获取全新 Cookie
3. 尝试其他微博账号

### 问题 3: 获取到空数据

**可能原因**:
- UID 错误或用户不存在
- 用户设置了隐私保护
- Cookie 权限不足

**解决方案**:
1. 验证 UID 是否正确
2. 检查用户是否有公开微博
3. 尝试其他公开用户的 UID

### 问题 4: 编译错误

**错误信息**: `The command could not be loaded... No .NET SDKs were found`

**解决方案**:
```bash
# 确保已安装 .NET 6.0 SDK
dotnet --version

# 如果未安装，下载并安装
# https://dotnet.microsoft.com/download/dotnet/6.0
```
